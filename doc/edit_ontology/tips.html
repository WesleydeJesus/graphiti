Tips

Here are listed a few common patterns seen in Graphiti ontologies.

Type pattern

Any object (that is a graph, a vertex or an edge) in Graphiti must belong to a given type. There can be as little as one type per category, for instance it is common to see only one type of connection between vertices. On the contrary vertices are often of different types, e.g. in dataflow applications you may encounter input and output ports, as well as instances.

"type" parameter

To model this behavior, Graphiti has a builtin "type" parameter that shall be associated with any object. This parameter should rarely be changed by the user, for different types often have completely different characteristics.
